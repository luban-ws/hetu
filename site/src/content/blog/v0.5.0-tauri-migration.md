# v0.5.0 — The Tauri Migration

We're excited to announce **Hetu v0.5.0**, the biggest architectural change since the project began. The entire desktop shell has been migrated from Electron to **Tauri 2**, with Git operations rewritten in **Rust**.

## Why Tauri?

Electron served us well, but came with trade-offs: large binary size (~150 MB), high memory usage, and bundling an entire Chromium instance. Tauri uses the platform's native webview, resulting in:

- **~10x smaller** binary (15 MB vs 150 MB)
- **~3x faster** startup time
- **Native** window chrome and OS integration
- **Rust** backend for Git operations — safer and faster than NodeGit

## What Changed

### Backend: Rust Git Core

All Git operations now run through Rust's `git2-rs` crate instead of NodeGit. This includes:

- Repository status and file listing
- Commit log and graph computation
- Diff generation
- Push, pull, fetch, and branch operations
- Credential storage via OS-native keychain APIs

### Frontend: Desktop Adapter

The Angular frontend now communicates through a `DesktopAdapter` abstraction layer that cleanly separates IPC concerns from business logic. This makes the code testable and keeps the door open for future platform targets.

### Settings & Persistence

A new `settings_store.rs` module provides atomic, JSON-based settings persistence at `~/Hetu/settings.json`. Repository history and the "last opened repo" state are now reliably restored on startup.

## Migration Guide

If you're building from source:

```bash
# You now need Rust installed
curl --proto '=https' --tlsv1.2 -sSf https://sh.rustup.rs | sh

# Then build as usual
npm install
npm run tauri:dev
```

The Electron build path (`npm run dev`) is still available as a legacy option but will be removed in a future release.
